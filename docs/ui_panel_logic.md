# 游戏主菜单面板跳转逻辑总结

## 一、主界面布局结构

主界面为所有功能的入口，采用**静态背景 + 居中控件**的布局：

| 区域 | 内容 | 交互类型 |
|------|------|----------|
| **顶部状态栏** | 资源显示（如金币 477）、**选项** 按钮 | 点击“选项”弹出中央面板 |
| **中央核心区** | 游戏标题、**开始** 按钮（主 CTA） | 点击“开始”弹出中央面板（模式选择/角色选择） |
| **底部功能区** | **收藏**、**增强**、**成就** 三个并行按钮 | 各自点击均弹出中央面板 |
| **底部信息区** | 版本号 |  |

---

## 二、核心设计原则：单一固定面板容器

整个主菜单的二级界面全部采用 **“同一个固定尺寸面板”** 进行承载，通过切换面板内部内容来实现不同系统的展示。

### 2.1 面板规格

- **位置**：屏幕正中央，绝对居中
- **尺寸**：固定宽高（所有系统共用同一尺寸，不允许自适应撑大或缩小）
- **层级**：浮于主界面之上，主界面仍然可见但处于不可交互状态（或加暗色遮罩）
- **动画**：快速弹出

### 2.2 返回按钮规范

- **位置**：固定于面板的 **右上方**（或面板内部的右上角）
- **行为**：点击后立即关闭当前面板，返回主界面
- **统一性**：所有面板共用同一套返回逻辑与位置规范

---

## 三、面板内容映射表

以下系统全部共用上述固定面板，仅填充内容不同：

| 触发按钮 | 面板内容 | 典型内部元素 |
|----------|----------|--------------|
| **选项 / Options** | 设置面板 | 音量滑块、语言选择、全屏开关、操作方式切换等 |
| **开始 / Start** | 游戏准备面板 | 模式选择（如剧情/无尽/挑战）、角色选择、难度选择、确认按钮 |
| **增强 / Power Up** | 成长/强化面板 | 永久属性升级树、消耗金币进行强化 |
| **成就 / Achievements** | 成就面板 | 成就列表（已完成/未完成）、进度条、解锁奖励提示 |
| **排行榜 / Leaderboard** | 排行榜面板 | 分数排行、玩家名次 |
| **联机模式 / Multiplayer** | 联机面板 | 创建房间、加入房间、房间列表、邀请码输入 |

## 五、设计优势与建议

### 5.1 为什么采用单一固定面板？

1. **一致性**：玩家无需适应不同尺寸和位置的弹窗，降低认知负担。
2. **开发效率**：只需维护一套面板容器代码，不同系统以组件/页面形式注入内容。
3. **扩展性**：新增系统（如后续 DLC 的“每日挑战”）无需调整 UI 框架，直接复用容器。
4. **响应式友好**：在分辨率变化时，只需保证中央面板居中且大小固定，布局稳定性强。

### 5.2 开发实现建议

```text
UI 层级结构建议：
├── Canvas (主画布)
│   ├── MainMenuBackground (主界面背景与按钮，Persistent)
│   ├── OverlayMask (半透明遮罩，面板打开时激活)
│   └── CentralPanelContainer (固定尺寸面板容器)
│       ├── ReturnButton (返回按钮，固定右上)
│       ├── PanelContentRoot (内容挂载点)
│       │   ├── SettingsContent (设置内容预制体)
│       │   ├── StartGameContent (开始游戏内容预制体)
│       │   ├── CollectionContent (收藏内容预制体)
│       │   ├── PowerUpContent (增强内容预制体)
│       │   ├── AchievementContent (成就内容预制体)
│       │   └── ... (其他系统内容)
│       └── PanelAnimator (打开/关闭动画控制器)
```

### 5.3 交互细节建议

- **打开面板时**：主界面按钮失活，显示遮罩，播放面板打开动画。
- **关闭面板时**：播放关闭动画，隐藏遮罩，恢复主界面按钮交互。
- **支持 ESC 键**：键盘按下 ESC 等效于点击返回按钮，关闭当前面板。
- **禁止多层嵌套**：所有面板平级切换，避免“面板上层面板”的复杂堆栈。若必须嵌套，仍应复用同一容器，替换内容而非叠加。

---

## 六、总结

该交互模式的核心是 **“主界面为锚点，中央固定面板为唯一出口”**。无论玩家点击哪个系统按钮，视觉焦点始终被引导至屏幕中央的同一区域内，右上角的返回按钮提供了一条稳定、可预期的退出路径。这种设计简洁、可扩展，非常适合功能模块较多但不需要复杂层级导航的独立游戏主菜单。
