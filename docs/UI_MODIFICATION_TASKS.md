# ForgeDuel UI 改造提示词

## 改造目标

优化子菜单的显示方式，解决 UI 割裂问题。

## 改造要求

### 1. 菜单显示方式
- **现状**：每个子菜单（升级商店、排行榜、成就等）是独立的 Scene，切换时跳转到新页面
- **目标**：子菜单直接在主菜单页面上叠加显示（类似 Overlay），而不是跳转到新页面

### 2. 返回键设计
- **位置**：固定在主菜单左上角，始终可见（不随子菜单切换消失）
- **状态**：
  - 默认状态：显示返回键图标，但不显示可点击的高亮效果
  - 打开子菜单后：返回键显示可点击的高亮颜色（金色边框、黄色文字）
  - 点击返回键：关闭当前子菜单，返回键恢复默认状态
- **样式**：使用提供的像素图图片作为返回键图标
- **层级**：返回键始终在最顶层（depth: 9999）

### 3. 实现思路

#### 方案 A：Overlay 面板方式
- 在 `MainMenuScene` 中管理所有子菜单面板的显示/隐藏
- 使用 `setVisible(true/false)` 控制面板显示
- 返回键的回调函数根据当前打开的子菜单返回到主页

#### 方案 B：容器 + 深度管理
- 将子菜单内容放入 `Phaser.GameObjects.Container`
- 设置容器的 depth 确保在主菜单之上
- 返回键作为主菜单的永久元素，不属于任何子菜单

### 4. 参考代码

现有返回键代码位于 `src/ui/createBackButton.js`，可参考其样式：
- 位置：x=60, y=36
- 尺寸：100x36
- 边框：金色 0xd4af37，hover 时变为黄色 0xfef08a
- 字体：Zpix，15px
- 层级：depth=9999

### 5. 需要修改的文件

1. `src/scenes/MainMenuScene.js` - 添加返回键管理逻辑
2. `src/ui/createBackButton.js` - 支持默认状态和激活状态
3. `src/scenes/UpgradeScene.js` - 改为面板叠加方式
4. `src/scenes/LeaderboardScene.js` - 改为面板叠加方式
5. 其他子菜单 Scene 同理

### 6. 注意事项

- 确保返回键的点击区域足够大，方便触摸操作
- 子菜单打开时，主菜单背景应该保持可见
- 动画效果应该柔和，避免生硬的切换

## 预期效果

打开子菜单 → 主菜单背景 + 子菜单面板叠加显示 + 返回键高亮可点击
关闭子菜单 → 只显示主菜单 + 返回键恢复默认状态
